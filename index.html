FlyRank ML Research Paper

Abstract

This paper presents the machine learning analysis performed during the FlyRank ML Internship. The study focuses on building and evaluating a machine learning model using production search data. This work is presented as a case study of FlyRank's content performance problem using public-safe language. The findings provide transparent evaluation and practical recommendations for content optimization.

The Random Forest model achieved 52% accuracy on the test split. The results show that the model captures some patterns from historical search and engagement data, but performance is limited by class imbalance and available data distribution.

Limitations and honest framing

This analysis identifies observed patterns between historical search performance and available engagement features. The model is intended for decision support and does not prove Google's ranking algorithm or guarantee future content performance.

The dataset contains imbalanced activity classes, which can affect model performance, especially for less frequent content categories. Results should be interpreted as directional insights for content improvement and prioritization.

Ranked Recommendations

Based on the model findings and observed search performance patterns, the following recommendations can support content prioritization decisions:

  1. Review low-performing content: Identify content with weak search activity and evaluate opportunities for updates, improved relevance, or better user engagement.
  2. Prioritize high-potential pages: Focus on content showing positive historical signals where optimization may improve discoverability.
  3. Monitor search performance trends: Regularly track changes in impressions, clicks, and engagement signals to identify content requiring attention.
  4. Use model outputs as decision support: Apply predictions as recommendations for review rather than guaranteed outcomes.
